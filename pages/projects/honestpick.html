프로젝트 개요

리뷰 서비스의 가장 큰 문제는 “안 가본 사람이 남기는 리뷰”였습니다.
HonestPick은 예약 또는 방문 이력이 있는 사용자에게만 리뷰 권한을 주는 방식으로 리뷰의 신뢰도를 높였습니다.
추가로 점주가 멤버십을 결제하면 매장이 우선 노출되도록 해 사용자-점주-관리자 모두에게 이득이 되는 구조를 목표로 했습니다.

  • 예약 → 방문처리 → 리뷰작성 으로 이어지는 플로우
  • 리뷰 1건에 최대 5개 이미지 첨부 (JSP MultipartRequest)
  • 관리자 페이지에서 매장 등록/비공개, 리뷰 관리

주요 기능

회원/인증

세션 기반 로그인, 권한 체크, 예약 이력과 연동

인증 리뷰

예약/방문 이력이 있는 사용자만 리뷰 작성 가능

다중 파일 업로드

파일 정보를 별도 테이블로 분리해 리뷰와 1:N 구성

관리자

매장 등록/비활성, 신고 리뷰 숨김 처리

주요 화면

기술 스택

  • Language : Java
  • Web : JSP / Servlet, JSTL
  • DB : Oracle
  • Infra : Tomcat 9
  • 외부 API : Kakao Map, Toss 결제(점주 멤버십)

주요 모듈

예약 리뷰 첨부파일 매장 관리자

DB 설계 포인트

리뷰와 첨부파일을 조인해 한 번에 가져오면 리뷰 하나가 파일 개수만큼 중복돼 내려오는 문제가 있어, DAO 단계에서 리뷰 1건 + 파일 리스트로 합쳐주는 구조를 사용했습니다.

  • REVIEW (review_id, member_id, store_id, content, rating, regdate ...)
  • REVIEW_FILE (file_id, review_id, org_name, save_name, size ...)
  • RESERVATION (resv_id, member_id, store_id, resv_date, status ...)
  • STORE (store_id, name, addr, tel, is_active ...)
