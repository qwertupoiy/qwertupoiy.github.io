프로젝트 개요

경제 뉴스는 용어도 어렵고 정보도 많아, 처음 접하는 사람은 무엇을 봐야 할지부터 막히는 경우가 많습니다.
Summarix는 경제 뉴스, 유튜브 댓글, 경제 지표 데이터를 모아 AI로 분석하고, 오늘의 경제 흐름을 한 화면에서 볼 수 있게 만든 웹서비스입니다.

분석 서버(FastAPI)와 서비스 서버(Spring Boot)를 분리한 이중 서버 구조로 설계해 ML/AI 파이프라인을 바꿔도 웹서비스는 그대로 유지될 수 있도록 했습니다.
저는 이 구조 안에서 특히 사용자 선호도 기반 개인화 도서/영상 추천 기능을 중심으로 백엔드를 구현했습니다.

  • 기간: 2025.09 ~ 2025.10
  • 구성: 팀 프로젝트 · Spring Boot(Web) + FastAPI(AI/Data)
  • 역할: 전체 데이터 흐름 설계 참여 및 개인화 추천 모듈 구현 중심 기여

주요 기능

경제 뉴스 수집 · 요약

외부 API에서 경제 뉴스를 주기적으로 수집하고, AI로 핵심 내용을 자동 요약합니다.

감성 · 핫토픽 분석

뉴스 본문과 유튜브 댓글을 분석해 긍·부정 감성, 주요 키워드, 급등 이슈를 추출합니다.

대시보드 시각화

키워드 트렌드, 감성 변화, 여론 분석 결과를 그래프·차트로 한 화면에 보여줍니다.

개인화 추천

MongoDB에 저장된 선호 데이터를 기반으로 경제 도서와 유튜브 영상을 개인별로 추천합니다.

주요 화면

기술 스택

  • Service : Spring Boot (Java)
  • Analysis : FastAPI (Python)
  • DB : MongoDB Atlas
  • Data : Pandas, Transformers
  • AI : GPT 기반 요약/챗봇, 감성 분석

주요 모듈

데이터 수집(뉴스/유튜브/지표) 전처리 · 분석 · 요약 감성·키워드 대시보드 개인화 도서 추천 개인화 영상 추천

담당 역할 & 기여

백엔드 관점에서 데이터가 수집 → 분석 → 사용자에게 도착하는 흐름을 함께 설계하고, 그 중 개인화 추천과 서버 간 연동 부분을 중점적으로 구현했습니다.

  • Spring Boot(Web) ↔ FastAPI(AI/Data) 이중 서버 구조 및 응답 포맷 설계에 참여
  • 뉴스/댓글/지표 분석 결과를 위한 DTO 및 JSON 응답 구조 정의
  • 회원가입 시 카테고리 선호도(Explicit) 설문 저장 로직 및 스키마 설계
  • 사용자 활동을 확장 가능하게 저장하기 위한 Implicit 선호 데이터 구조 설계
  • MongoDB에 사용자 선호 데이터를 저장하고, 이를 기반으로 경제 도서·유튜브 영상 추천 기능 구현

기술적 고민 & 인사이트

  • AI 분석 서버와 웹 서버를 분리해, 모델·파이프라인이 바뀌어도 서비스는 안정적으로 유지되는 구조를 경험했습니다.
  • 개인화 추천을 위해 문서형 DB(MongoDB)를 사용하고, 선호 데이터를 계층형 구조로 설계해 향후 활동 로그까지 확장 가능하게 했습니다.
  • “AI가 분석한 결과”를 보여주는 데서 끝나지 않고, 어떤 책·영상으로 다음 단계를 공부할지까지 연결하는 사용자 여정을 고민했습니다.

데이터 & 분석 포인트

서로 다른 소스의 데이터를 결합해 경제 흐름을 입체적으로 보여주는 것을 목표로 했습니다.

  • 뉴스: 경제 카테고리 기사 수집(제목, 본문, 언론사, 발행일 등)
  • 유튜브: 댓글·좋아요 기반 감성 및 여론 분석
  • 경제 지표: ECOS/FRED 등에서 주요 지표를 가져와 추이 그래프로 제공
  • 시각화: 키워드 랭킹, 감성 비율, 트렌드 그래프로 “지금 시장 분위기”를 직관적으로 표현
