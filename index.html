AI 기반 경제 뉴스 분석 웹서비스 Spring Boot + FastAPI

2025.09 ~ 2025.10 역할: 백엔드 설계 및 데이터 연동

경제 뉴스를 수집·요약하고 감성 분석/핫토픽을 시각화해 보여주는 대시보드 서비스입니다. FastAPI로 수집·분석한 데이터를 Spring Boot에서 받아서 화면(Thymeleaf)까지 전달하는 백엔드 아키텍처를 직접 구성했습니다.

분석 서버(FastAPI)와 서비스 서버(Spring Boot)를 분리한 이유는, 모델 로직이나 데이터 파이프라인을 바꿔도 서비스 인증/세션 로직이 깨지지 않게 하기 위함입니다. 즉, 분석 쪽은 실험적으로 계속 바뀌어도 되고, 서비스 쪽은 안정적으로 유지할 수 있도록 계층을 나눴습니다.

  • Spring Boot ↔ FastAPI 간 연동 구조 설계 (WebClient 사용)
  • MongoDB 기반 사용자 선호도/관심 카테고리 저장 구조 설계
  • 회원별 개인화 추천(도서·영상) 기능 구현
  • 로그인 및 세션/쿠키 기반 자동 로그인 흐름 구현
  • 실시간 키워드 랭킹, 유튜브 댓글 감성 분석 결과를 대시보드에 노출
개인화 추천 기능
사용자가 관심 있게 본 키워드/카테고리를 MongoDB에 저장하고, 해당 정보를 기반으로 경제 관련 도서·영상 추천을 제공합니다. 이 로직은 단순 인기순 추천이 아니라, “그 사용자가 본 맥락”을 반영하도록 설계했습니다.
  • 기술: Spring Boot, FastAPI(Python), MongoDB Atlas, Pandas, Transformers
  • 성과: 수집→분석→시각화 파이프라인과 서비스 로직을 단일 구조로 통합

시연 영상

대시보드 예시 화면. 실시간 경제 키워드, 감성 추이, 유튜브 의견 분석 결과가 한 화면에 정리됩니다.

HonestPick - 음식점 리뷰 및 예약 웹사이트 JSP / Servlet

2025.05 ~ 2025.06 역할: 백엔드 전반 구현

예약 기반 리뷰 시스템으로, 실제로 방문/예약한 사용자만 리뷰를 남길 수 있게 하여 허위 리뷰를 줄이는 것을 목표로 한 웹서비스입니다. 결제, 지도, 관리자 기능까지 실제 서비스 형태에 가깝게 구현했습니다.

특히 리뷰 작성은 '해당 매장을 실제로 예약했던 사용자'만 가능하도록 제한해 신뢰도 높은 리뷰만 쌓이도록 했고, 점주는 Toss 결제 API를 통해 멤버십을 결제하면 상단 노출 우선순위를 확보할 수 있도록 비즈니스 로직을 구현했습니다.

이 서비스는 백엔드 단일 애플리케이션(JSP/Servlet 기반) 위에서 인증(세션/쿠키), 결제, 위치 기반 안내(Kakao 지도 API), 관리자 관제까지 돌아가도록 구성했기 때문에 “상용 서비스에 가까운 전체 흐름”을 직접 다뤄봤다는 점에 의미가 있습니다.

  • 예약 여부 검증 후 리뷰 허용(허위 리뷰 방지 로직)
  • Toss 결제 API로 점주 멤버십 결제 → 상단 노출 우선권 부여
  • Kakao 지도 API로 매장 위치/정보 시각화
  • 세션/쿠키 기반 자동 로그인 및 유지 흐름 구현
  • Oracle DB 기반 매장/회원/리뷰 CRUD 처리 및 SQL 설계
데이터 & 운영 관점
Oracle 스키마를 직접 설계하여 매장, 사용자, 리뷰, 결제이력 테이블을 분리하고 JOIN 성능과 검색 편의성을 함께 고려했습니다. 이 구조 덕분에 관리자 페이지에서 “문제 리뷰 찾기 / 특정 점주 매장만 모아서 보기 / 결제 히스토리 확인” 같은 운영 기능을 빠르게 조회할 수 있습니다.
  • 기술: JSP, Servlet, Oracle DB, Tomcat 9, Kakao Map API, Toss 결제 API
  • 성과: 단일 서버 환경에서도 결제/지도/리뷰/관리자 기능까지 포함한 실제 서비스 흐름 구현

시연 영상

메인 화면. 매장 목록, 위치, 예약/후기 흐름을 한 번에 보여주도록 구성했습니다.

다음 목표 향후 성장 방향

  • Spring 기반 인증·권한 관리(회원/관리자 역할 분리) 고도화
  • 추천 로직을 단순 규칙 기반에서 행동 데이터(클릭, 체류시간 등) 기반으로 확장
  • 데이터 파이프라인 자동화 및 배치 스케줄링 안정화
  • 데이터 엔지니어링 관점에서의 저장 구조/ETL 최적화 경험 강화

현재 관심사는 "데이터를 어떻게 모으고 구조화해서 사용자에게 의미 있는 형태로 전달할 것인가"입니다. 백엔드 개발 경험을 기반으로, 궁극적으로는 안정적인 데이터 파이프라인과 맞춤형 서비스를 운영할 수 있는 데이터 엔지니어로 성장하는 것을 목표로 하고 있습니다.
